🛠 Техническое задание: Оптимизация платежных шлюзов и UI/UX системы донатов
Цель задачи: Упростить процесс оплаты для пользователей, разделить логику обработки фиатных (UAH) и криптовалютных платежей, внедрить автоматический конвертер валют и обновить интерфейс страницы ожидания оплаты.

1. Изменение логики платежных шлюзов (Backend)
1.1. Оплата через Crypto Bot (Crypto Pay API)

Отказ от системы чеков: При выборе Crypto Bot внутренний «чек» (таймер/блокировка суммы) больше не создается.

Интеграция: Использовать нативные инвойсы Crypto Pay API. Документация: https://help.send.tg/en/articles/10279948-crypto-pay-api

Триггер успеха: Ожидаем webhook/callback от Crypto Bot. Как только крипта поступает — донат автоматически считается успешным, сообщение публикуется.

1.2. Оплата через TON (Ton Pay)

Отказ от системы чеков: Аналогично Crypto Bot, внутренний чек не создается.

Интеграция: Использовать TON Pay.

UX авторизации: Реализовать возможность оплаты не только через Tonkeeper, но и напрямую через внутренний кошелек Telegram (Wallet).

1.3. Оплата через Monobank (UAH)

Сохранение чеков: Для фиатных оплат через Монобанк система генерации чека и таймера (10 минут) сохраняется.

Интеграция Webhook: Настроить Webhook на хосте kruyz.fun. Документация: https://api.monobank.ua/docs/index.html

API Ключ: Использовать токен: ul3jSER7JO9KNeXc2qx5gnVTCCVmqDFdhPbIfCRfYgKg

Логика валидации платежа (Fuzzy Matching):

Стандартная: Пользователь переводит нужную сумму и указывает Код чека в комментарии к платежу. Webhook ловит транзакцию -> донат успешен.

Edge Case (Забыл/не указал код): Если Webhook ловит платеж без нужного комментария, система проверяет активные чеки за последние 10 минут. Если существует только один активный чек на эту точную сумму, система автоматически привязывает этот платеж к данному чеку и успешно проводит донат (с суммой и сообщением пользователя).

2. Обновление UI/UX (Frontend)
2.1. Главное меню доната (Конвертер валют)

Динамическая конвертация: Добавить калькулятор реального времени.

Пример 1: Минимальный донат 1 UAH. При переключении способа оплаты на TON, в поле суммы автоматически подставляется эквивалент 1 UAH в TON по текущему курсу.

Пример 2: Пользователь вводит "1 TON" и переключается на UAH -> поле обновляется до актуального курса (например, 59.20 UAH).

Информационный текст: Над полем выбора суммы всегда должна отображаться актуальная подсказка: (Сума [ N usdt/ton] = [N] [грн]).

2.2. Страница ожидания оплаты (для Monobank)

Инструкция по коду: Код чека должен дублироваться в самом низу страницы (возле поля "Повідомлення"). Добавить пояснительный текст, чтобы пользователь обязательно указал этот код в комментарии к переводу в UAH.

Редирект при оплате: После нажатия «Надіслати» (Оплатить), пользователя должно перекидывать в новой вкладке на банк (или открывать приложение Telegram/Tonkeeper/Monobank).

Обновленные элементы управления: 3 массивные кнопки:

Оплатити (Перейти) 2. Перевірити / відмінити (назад) 2.3. Маршрутизация и очистка состояния

Истечение времени: Если таймер чека истек, возвращаем назад к созданию доната.

Успешная оплата: После успешной оплаты чистим все введенные данные доната (сумма, сообщение), кроме поля "Ім'я".
