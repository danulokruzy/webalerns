🛠 ТЗ: Обновление Панели Стримера (Раздел Settings & Donations)
1. Архитектура раздела "Settings"
Вместо трех разных вкладок (Setup, Connections, Settings) создается единая страница с логическим разделением на блоки. Это избавит пользователя от лишних кликов.

Основные блоки на странице:

Интеграции (Connections):

Привязка TikTok (через Session ID или логин).

Привязка Minecraft (IP сервера, порт, API-ключ).

Привязка Monobank (Token для доступа к выпискам по банкам).

Настройка Telegram Bot (Токен для уведомлений).

Платежные данные:

UAH (Monobank): Поле для ссылки на банку.

Crypto (CryptoBot): API Token приложения.

TON (Direct): Поле для адреса кошелька (для генерации QR-кодов и прямых переводов).

Общие настройки:

Префикс кода чека (например, DON-).

Лимиты доната (Мин/Макс сумма в грн).

Смена пароля Dashboard.

2. Логика работы с TON
Больше никаких статичных ссылок TonPay URL.

Механика: При выборе оплаты через TON, система должна генерировать уникальный инвойс или transfer link.

Для PC: Вывод на экран QR-кода. Пользователь сканирует его телефоном (Tonkeeper/MyTonWallet) и подтверждает транзакцию.

Для Mobile: Прямая кнопка-ссылка (deep link), открывающая установленный кошелек.

3. Автоматизация курсов валют
Полностью удаляем поля ручного ввода Fallback USDT/TON.

Реализация: Backend раз в 5–10 минут опрашивает публичные API (например, Binance P2P API, CoinGecko или WhiteBit).

Приоритет: Для пары USDT/UAH берем актуальный курс продажи на Mono/P2P, чтобы стример не терял на разнице при конвертации крипто-донатов.

4. Система "Donations" (История и мониторинг)
Раздел "Donations" теперь становится единым журналом всех входящих средств.

Источники данных:

Webhooks: Мгновенное получение данных от CryptoBot и TonPay.

Mono API: Постоянный мониторинг выписки (через Webhook монобанка). Если на банку падает любая сумма — она тут же отображается в списке.

Отображение в таблице:

ID/Чек: (например, DON-12345).

Источник: Иконка (Моно, TON, CryptoBot).

Сумма: В оригинальной валюте и эквивалент в UAH (по курсу на момент оплаты).

Статус: Оплачено / В ожидании / Ошибка.

Текст доната: Если передан через мета-данные или комментарий к платежу.

5. UI/UX Правки (по макету)
Боковое меню: Удаляем пункты "Setup" и "Connections". Оставляем "Actions", "Donations", "Widgets", "Logs" и объединенный "Settings".

Стиль: Сохраняем Glassmorphism. Все поля ввода в блоке Settings должны иметь плейсхолдеры с примерами (например, https://send.monobank.ua/jar/...).

Кнопка сохранения: Должна быть плавающей (sticky) или находиться внизу страницы, чтобы не скроллить каждый раз.
